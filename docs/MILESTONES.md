# Milestones

Note: One Codex run per phase is the recommended strategy.

| Phase | Scope | Acceptance Criteria | Effort |
| --- | --- | --- | --- |
| Phase 0: MVP spec + scope freeze | Confirm scope, data model, and validation rules | PLAN.md and DECISIONS.md approved | XS |
| Phase 1: Project skeleton + routing | App shell and navigation scaffolding | App opens and routes between placeholder screens | S |
| Phase 2: IPC contract + domain models + validation design | Define API surface and validation rules | IPC contract documented and reviewed | S |
| Phase 3: SQLite schema + migrations + repositories (design only) | Schema, migrations, repository interfaces | Storage design reviewed and approved | M |
| Phase 4: Quiz list + quiz editor UI (design/steps only) | Quiz list and editor UX design | UI flow documented with screens and actions | M |
| Phase 5: Question editor UI + validation (design/steps only) | Question and option UX design | Validation and UX behavior documented | M |
| Phase 6: Player + results (design/steps only) | Player flow and results design | Player flow documented end-to-end | M |
| Phase 7: Packaging + smoke tests | Packaging plan and manual tests | Packaging plan ready for implementation | S |
